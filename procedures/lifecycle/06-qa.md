# Procedure: 06-qa

Run `/impeccable` critique on the built app. The gate requires ≥ 7/10 with zero P0 findings.

If score < 7/10 or any P0: fix and rerun. Do not waive. The impeccable critique result is the scored artifact proving the design-quality gap (3.6/10 Entry 001) has been closed.

**Evidence required:** `apps/<app>/proof/qa-eval-result.json` containing the impeccable critique result with score ≥ 7 and an empty P0 list.
